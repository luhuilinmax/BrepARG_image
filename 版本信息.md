## 2026-05-11 补充修复：DDP 初始化顺序

第一次修改后，验证集只在 `rank 0` 加载，但加载发生在 `VQVAETrainer` 初始化之前。由于 `VQVAETrainer` 内部才会初始化 DDP process group，非 `rank 0` 会先进入 DDP 初始化并等待，而 `rank 0` 仍在加载验证集，最终可能导致其他 rank 超时退出。

```
补充修复前的执行流程
rank 0:
  加载 train_dataset
  加载 val_dataset，很慢，日志显示 21 分钟才到 12%
  还没进入 VQVAETrainer
  还没进入 DDP 初始化

rank 1/2/3/4:
  加载 train_dataset
  跳过 val_dataset
  进入 VQVAETrainer
  初始化 DDP / 包装 DDP 模型
  等 rank 0 一起参与通信
```

本次补充修复调整为：

```text
所有 rank 先加载训练集
所有 rank 先初始化 VQVAETrainer / DDP process group
rank 0 再加载验证集并挂到 trainer 上
其他 rank 在 barrier 等待 rank 0 完成验证集加载
```

同时将 `dist.init_process_group` 的 timeout 调整为 4 小时，避免验证集加载和 rank 0 验证耗时较长时，其他 rank 在 barrier 中过早超时。

训练循环中也增加了验证和保存后的同步：

```text
rank 0 执行验证后，所有 rank barrier
rank 0 保存模型后，所有 rank barrier
```

这样可以避免非 `rank 0` 在 `rank 0` 验证或保存期间提前进入下一轮 DDP 训练。

## 2026-05-28 补充：VQVAE 单样本 round-trip CAD 重建验证脚本

新增脚本：

```text
reconstruct_vqvae_sample.py
```

该脚本用于在没有 AR 权重时，验证自己训练的 VQVAE 是否能把真实 ABC 样本的 face/edge 几何编码成 geometry tokens，再解码回可构造 CAD 的几何。它不是完整的无条件 CAD 生成脚本，因为 face 数量、edge 数量、bbox 和 topology 仍然来自真实样本；真正从 `START` token 自回归生成完整 CAD 序列仍然需要 AR 权重。

本脚本的验证流程：

```text
真实 ABC pkl
  -> 读取 surf_ncs / edge_ncs / surf_bbox_wcs / edge_bbox_wcs / edgeFace_adj
  -> 用 VQVAE encoder + quant_conv + quantize 得到 face/edge geometry tokens
  -> 用 VQVAE decoder 解回 surf_ncs_recon / edge_ncs_recon
  -> 计算 face MSE / edge MSE
  -> 组装 holistic token sequence
  -> 调用 reconstruct_cad_from_sequence()，没修改这个函数
  -> bbox 反量化、VQVAE decoder、joint optimization、OpenCascade construct_brep/sew
  -> 输出 npz、sequence、ordering、metrics、STEP/STL
```

为了让验证逻辑尽量接近原始 `2sequence.py` / 论文的序列组织方式，脚本已补齐：

```text
1. face block 顺序：按 2sequence.py 的 DFS from core 逻辑排序；
2. edge block 顺序：按 MAX-IDX-A，即 (max(face), min(face)) 字典序排序；
3. face index token：按 cyclic offset 做 re-index；
4. 增加 --seed 和 --reindex_offset，便于复现实验和调试；
5. 输出 ordering json，记录 face_order、edge_order、ordered_edge_face_pairs、face_index_map、cyclic_offset。
```

需要注意：该脚本虽然组装了和 AR 训练数据一致风格的 holistic sequence，并复用了 `reconstruct_cad_from_sequence()` 的后半段，但 sequence 中的 bbox/topology/face-edge 数量来自真实样本，geometry tokens 来自 VQVAE encode 真实几何。因此它验证的是 VQVAE geometry tokenization / detokenization 和 B-rep 后处理可构造性，不验证 AR 的从无到有生成能力。
