## 2026-05-11 补充修复：DDP 初始化顺序

第一次修改后，验证集只在 `rank 0` 加载，但加载发生在 `VQVAETrainer` 初始化之前。由于 `VQVAETrainer` 内部才会初始化 DDP process group，非 `rank 0` 会先进入 DDP 初始化并等待，而 `rank 0` 仍在加载验证集，最终可能导致其他 rank 超时退出。

```
补充修复前的执行流程
rank 0:
  加载 train_dataset
  加载 val_dataset，很慢，日志显示 21 分钟才到 12%
  还没进入 VQVAETrainer
  还没进入 DDP 初始化

rank 1/2/3/4:
  加载 train_dataset
  跳过 val_dataset
  进入 VQVAETrainer
  初始化 DDP / 包装 DDP 模型
  等 rank 0 一起参与通信
```

本次补充修复调整为：

```text
所有 rank 先加载训练集
所有 rank 先初始化 VQVAETrainer / DDP process group
rank 0 再加载验证集并挂到 trainer 上
其他 rank 在 barrier 等待 rank 0 完成验证集加载
```

同时将 `dist.init_process_group` 的 timeout 调整为 4 小时，避免验证集加载和 rank 0 验证耗时较长时，其他 rank 在 barrier 中过早超时。

训练循环中也增加了验证和保存后的同步：

```text
rank 0 执行验证后，所有 rank barrier
rank 0 保存模型后，所有 rank barrier
```

这样可以避免非 `rank 0` 在 `rank 0` 验证或保存期间提前进入下一轮 DDP 训练。

## 2026-05-28 补充：VQVAE 单样本 round-trip CAD 重建验证脚本

新增脚本：

```text
reconstruct_vqvae_sample.py
```

该脚本用于在没有 AR 权重时，验证自己训练的 VQVAE 是否能把真实 ABC 样本的 face/edge 几何编码成 geometry tokens，再解码回可构造 CAD 的几何。它不是完整的无条件 CAD 生成脚本，因为 face 数量、edge 数量、bbox 和 topology 仍然来自真实样本；真正从 `START` token 自回归生成完整 CAD 序列仍然需要 AR 权重。

本脚本的验证流程：

```text
真实 ABC pkl
  -> 读取 surf_ncs / edge_ncs / surf_bbox_wcs / edge_bbox_wcs / edgeFace_adj
  -> 用 VQVAE encoder + quant_conv + quantize 得到 face/edge geometry tokens
  -> 用 VQVAE decoder 解回 surf_ncs_recon / edge_ncs_recon
  -> 计算 face MSE / edge MSE
  -> 组装 holistic token sequence
  -> 调用 reconstruct_cad_from_sequence()，没修改这个函数
  -> bbox 反量化、VQVAE decoder、joint optimization、OpenCascade construct_brep/sew
  -> 输出 npz、sequence、ordering、metrics、STEP/STL
```

为了让验证逻辑尽量接近原始 `2sequence.py` / 论文的序列组织方式，脚本已补齐：

```text
1. face block 顺序：按 2sequence.py 的 DFS from core 逻辑排序；
2. edge block 顺序：按 MAX-IDX-A，即 (max(face), min(face)) 字典序排序；
3. face index token：按 cyclic offset 做 re-index；
4. 增加 --seed 和 --reindex_offset，便于复现实验和调试；
5. 输出 ordering json，记录 face_order、edge_order、ordered_edge_face_pairs、face_index_map、cyclic_offset。
```

## 2026-06-07 准备开展：AR 实验分层验证

当前状态梳理：

- `reconstruct_vqvae_sample.py` 已经说明当前 VQVAE 权重在真实样本上具备 round-trip 几何编码/解码能力，并且经过后处理后可以导出 valid CAD。
- 现在效果差的实验来自 AR 权重驱动的 `generate_brep.py` 随机采样，即真正的 free-running generation。
- 当前最需要区分的是：AR 是“在真实前缀下就没学会 next-token prediction”，还是“在真实前缀下还可以，但一旦自由滚动就迅速误差累积”。

因此准备把后续实验拆成三层，而不是继续只看最终随机采样结果：

### 1. VQVAE round-trip

目的：

- 继续作为底层几何 token 化与后处理可构造性的基线。

说明：

- 该实验不依赖 AR 权重。
- 真实 face/edge 数量、bbox、topology 来自样本本身。
- 主要输出是 MSE、sequence、中间数据、STEP/STL、`brep_valid`。

### 2. AR strict teacher-forced evaluation

目的：

- 使用已经训练出来的 AR 权重，在真实前缀条件下评估 next-token prediction 能力。

计划观察：

- overall loss / accuracy / perplexity
- 每个位置是否预测正确
- 不同 token 类型的表现
- face 段、edge 段、`SEP/END` 等结构位置的错误分布

注意：

- 该实验的主要输出是 token-level 诊断结果，而不是最终生成 CAD。
- 严格 teacher-forced 的逐位置预测通常不应直接硬拼成正式生成序列去评价 CAD。

### 3. AR free-running generation

目的：

- 使用同一份 AR 权重，从 `[START]` 开始自由采样，评估真实生成能力。

计划观察：

- 生成序列长度
- 是否能成功反解成 CAD
- `brep_valid`
- 成功率低的主要失败模式

### 4. 后续准备补充：AR partial teacher forcing

目的：

- 在真实前缀和自由生成之间做桥接实验，定位模型从哪个结构边界开始崩。

可能切法：

- 只给 `[START]`
- 给真实第一个 face block
- 给完整真实 face 段直到 `[SEP]`
- 给前若干 edge block

预期：

- 该实验最终会得到“真实前缀 + 生成后缀”的完整自洽序列，因此适合反解成 CAD 并比较不同切换点下的成功率。

本次文档准备阶段先补充实验定义与目标，不在这里记录具体结果；等后续真正实现 `teacher-forced eval` / `partial teacher forcing` 脚本并跑实验后，再继续更新本节。

  判断当前这份小数据 AR 权重，在真实 sequence 条件下的 token 建模能力到底如何。

  输入：

  - 你现有的 sequence_file
  - 你已经训练出来的 AR 权重
  - 优先评估 val split
  - 如有必要，再补少量 train split 做过拟合对照

  做法：
  对每条真实序列，整条送进 AR，计算每个位置的 next-token 预测质量。这个本质上和验证 loss 接近，但
  要做更细的拆分统计。

  重点不是只看一个平均 loss，而是把结果拆开看。

  这一轮要重点关注的指标

  1. 全局指标

  - val loss
  - perplexity
  - next-token accuracy
  - top-5 accuracy，如果方便就一起算

  作用：
  先看这个 AR 至少有没有学到基本分布。

  2. 按 token 类型分组
     建议至少拆成：

  - face_index tokens
  - bbox tokens
  - surface geometry tokens
  - edge geometry tokens
  - SEP
  - END

  作用：
  看问题主要出在哪类 token。
  如果 geometry token 还行，但 face_index 和 SEP/END 很差，说明结构建模更弱。
  如果 face_index 还行但 geometry token 很差，说明几何 token 序列本身难学。

  3. 按结构区段分组
     建议至少拆成：

  - face blocks 区段
  - SEP 附近
  - edge blocks 区段
  - 序列末尾收尾区段

  作用：
  看错误是不是在进入 edge 段后明显变差，这对你这个任务非常关键。

  4. 按序列位置统计
     例如按归一化位置或分桶统计：

  - 前 10%
  - 中间 40%
  - 后 50%
    或更细一点。

  作用：
  看是不是越往后越崩。
  如果前半段还不错，后半段明显恶化，很像长程依赖和误差积累前兆。

  5. 按样本复杂度分组
     按样本的：

  - face 数量
  - edge 数量
  - sequence length

  分组统计 teacher-forced 指标。

  作用：
  看模型是不是只会简单 CAD，不会复杂 CAD。
  这和你现在的直观观察要互相验证。

  6. 单样本级可视化诊断
     建议抽几条：

  - teacher-forced 表现最好的样本
  - 中等样本
  - 最差样本
  - 复杂样本

  输出：

  - 真实 token
  - 预测 token
  - 错误位置
  - 错误 token 类型
  - 错误从哪里开始连续出现

  作用：
  这个比平均指标更有解释性，汇报也更有说服力。

  这一步做完以后怎么解释结果

  如果出现下面几种情况：

  - teacher-forced overall 很差
    说明 AR 本身还没学好，先别纠结 free-running 采样参数。

  - teacher-forced overall 还行，但 edge/token 结构段很差
    说明问题集中在后半段拓扑和连接关系。

  - teacher-forced 在简单样本好、复杂样本差
    说明数据规模和模型容量不足，或者复杂结构长程建模弱。

  - teacher-forced 不错，但 free-running 很差
    说明 exposure bias 明显，下一步就该做 partial teacher forcing。

  实验 1 完成后要产出的结果
  建议最少产出这些文件或表格：

  - 一个总表：
      - overall loss
      - perplexity
      - overall accuracy

  - 一个 token-type 表
  - 一个 structure-stage 表
  - 一个 complexity-bucket 表
  - 几条典型样本的错误对比

  这一步通常不以 CAD 反解为主，不建议把 strict teacher-forced 预测硬拼成完整 CAD 结果。


###   实验 2：AR partial teacher forcing
  这是第二步，不是第一步。

  目标：
  定位模型从哪个结构边界开始自由生成会崩。

  做法：
  给真实前缀，到某个切换点后，后面全由模型自由生成。

  对你的 sequence，我建议按结构切，不按固定 token 数切。

  建议至少做 4 组：

  1. Prefix-0

  - 只给 [START]
  - 这就是 free-running baseline

  2. Prefix-face1

  - 给真实第一个完整 face block
  - 后面自由生成

  3. Prefix-all-face

  - 给完整真实 face 段直到 SEP
  - 只让模型生成 edge 段和结尾

  4. Prefix-face+few-edge

  - 给完整 face 段 + 前若干真实 edge block
  - 再自由生成剩余 edge 和结尾

  如果后面需要，再补固定长度前缀版本作为辅助。

  这一步要关注的指标

  1. 序列级合法性

  - 是否生成出完整序列
  - 是否出现异常提前 END
  - 是否超长
  - 是否 token 越界

  2. 结构合法性

  - face block 数量是否合理
  - edge block 数量是否合理
  - SEP 是否出现在合理位置
  - face index 是否超范围或混乱

  3. CAD 重建结果

  - 是否成功反解
  - brep_valid
  - STEP/STL 是否成功写出
  - 后处理失败类型

  4. 随切换点变化的成功率
     这是这一步最核心的指标。

  要看：

  - 只给 [START] 成功率
  - 给第一个 face block 后成功率
  - 给完整 face 段后成功率
  - 给部分 edge 后成功率

  如果给完整 face 段后成功率显著上升，说明主要问题在 face 生成阶段。
  如果给完整 face 段后仍然差，说明 edge/topology 段也很弱。

  5. 复杂度恢复能力
     对同一批简单/复杂样本，比较不同 prefix 条件下能否撑住复杂结构。  

  实验 2 完成后要产出的结果
  建议做一个对照表：

  - prefix 类型
  - sequence 完整率
  - CAD 重建成功率
  - brep_valid 率
  - 平均 face/edge 数
  - 主要失败模式

  这一步非常适合做成你后续和导师讨论的核心实验。

###  实验 3：free-running generation 复查
  第三步再回到纯生成。

  这时候不只是看“它很差”，而是带着前两步结论回来看：

  - 如果 strict TF 就差，那 free-running 差是必然的
  - 如果 strict TF 还行、partial TF 也能救回来一部分，而 free-running 很差，说明问题主要是
    rollout 漂移

  - 如果 partial TF 给到 face 段后明显改善，说明 face 段是最关键瓶颈

  最终建议的详细执行方案

  阶段 A：先做 strict teacher-forced eval
  输出：

  - overall loss / ppl / acc
  - token-type breakdown
  - structure-stage breakdown
  - complexity breakdown
  - 几个单样本错误分析

  成功标准：

  - 先搞清楚 AR 到底有没有学会

  阶段 B：再做 partial teacher forcing
  输出：

  - 4 组 prefix 条件下的完整序列
  - 对应 CAD 重建结果
  - prefix 条件和成功率的关系

  成功标准：

  - 搞清楚模型从哪一段开始自由生成会崩

  阶段 C：回头复查 free-running
  输出：

  - 与 A/B 的对照结论

  成功标准：

  - 形成完整解释链，不再只是“随机采样效果差”

  我建议你现在就先做的具体实验
  第一轮只做下面这一个最小闭环：

  1. 在 val split 上跑 strict teacher-forced eval
  2. 统计：

  - overall loss
  - perplexity
  - overall accuracy
  - face 段 vs edge 段 accuracy
  - 按 sequence length 分桶 accuracy

  3. 抽 10 条复杂样本做单样本错误分析
