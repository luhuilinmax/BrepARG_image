# 数据规模限制说明

本文档记录 `BrepARG_m_ddp_fix` 中和 CAD 样本规模有关的限制：哪些阶段会按 face/edge 数量过滤样本，哪些阶段不会，以及这些限制和当前 VQVAE 重建实验的关系。

## 1. 原始 STEP 解析阶段

原始 STEP 转 `.pkl` 的脚本中有 face 数量上限，但不是 `50`，而是 `200`。

代码位置：

```text
process_data/process_brep.py:12-14
process_data/process_brep.py:140-141
process_data/process_brep_order.py:12-14
process_data/process_brep_order.py:140-141
```

关键逻辑：

```python
MAX_FACE = 200

if len(list(solid.faces())) > MAX_FACE:
    return None
```

含义：

- 如果原始 STEP 中 face 数量超过 `200`，解析阶段会直接跳过该模型。
- 这里没有看到对应的 `MAX_EDGE = 150` 过滤。
- 所以这个阶段的限制是“非常粗的 STEP 解析保护”，不是 AR 训练用的 `max_face=50/max_edge=150`。

## 2. VQVAE 训练数据阶段

VQVAE 训练使用的是单个 face / 单条 edge 的 NCS 几何数据，而不是完整 CAD 的 holistic token sequence。因此 VQVAE 数据准备阶段通常不会按每个 CAD 的 `max_face=50/max_edge=150` 过滤。

相关代码位置：

```text
process_data/deduplicate_se_data.py:84-123
prepare_val_cache.py:18-39
dataset.py:31-93
dataset.py:95-165
dataset.py:167-229
```

这些代码主要做：

- 从每个 `.pkl` 中读取 `surf_ncs` 或 `edge_ncs`。
- 将所有 face NCS / edge NCS 展开、去重、堆叠或缓存。
- VQVAE 训练时从 `SurfData`、`EdgeData`、`CombinedData` 中取单个几何元素训练。

例如 `deduplicate_se_data.py` 的核心逻辑是遍历样本中的 `surf_ncs` 或 `edge_ncs`：

```python
key = "edge_ncs" if args.mode == "edge" else "surf_ncs"
arrs = data[key]
for bit_repr, real_val in zip(bits_arr, arrs):
    ...
    unique_data.append(real_val)
```

`prepare_val_cache.py` 也是把 split 中所有样本的指定字段 `vstack` 到一起：

```python
if field_name in data:
    arr = data[field_name]
    arrays.append(arr)

cached = np.vstack(arrays)
```

含义：

- VQVAE 训练数据不直接关心“一个 CAD 有多少 face / edge”。
- 它关心的是每个 `surf_ncs` 是否是 `(32, 32, 3)`，每个 `edge_ncs` 是否是 `(32, 3)`。
- 所以 `max_face=50/max_edge=150` 不是 VQVAE 训练阶段的主要过滤条件。

## 3. AR 序列预处理阶段

`50/150` 的主要限制来自 AR holistic token sequence 预处理脚本 `2sequence.py`。

代码位置：

```text
2sequence.py:811-819
2sequence.py:708-716
```

参数默认值：

```python
parser.add_argument('--max_face', type=int, default=50)
parser.add_argument('--max_edge', type=int, default=150)
```

实际过滤逻辑：

```python
if len(surf_ncs) == 0 or len(edge_ncs) == 0:
    return None

if len(surf_ncs) > int(self.args.max_face):
    return None
if len(edge_ncs) > int(self.args.max_edge):
    return None
```

含义：

- 当完整 CAD 样本的 face 数量超过 `max_face` 时，不生成 AR 训练 sequence。
- 当完整 CAD 样本的 edge 数量超过 `max_edge` 时，也不生成 AR 训练 sequence。
- 默认配置下就是 `face > 50` 或 `edge > 150` 的样本会被过滤掉。

因此，`max_face=50/max_edge=150` 是 AR 训练数据构造阶段的样本规模上限。

## 4. Face Index Token 容量限制

除了 AR 预处理的 `max_face`，还有一个更底层的 token 词表容量限制：`face_index_size=50`。

代码位置：

```text
config.json:7-10
2sequence.py:208-225
reconstruct_vqvae_sample.py:171-187
reconstruct_vqvae_sample.py:201-206
```

`config.json` 中 ABC 配置：

```json
"abc": {
  "face_index_size": 50,
  "se_codebook_size": 8192,
  "bbox_index_size": 2048
}
```

`2sequence.py` 中也固定了 face index token 的范围：

```python
self.face_index_size = 50
self.face_index_offset = 0
self.se_token_offset = self.face_index_offset + self.face_index_size
```

当前单样本 VQVAE 重建脚本 `reconstruct_vqvae_sample.py` 会检查 face 数量是否超过词表容量：

```python
num_faces = len(surf_tokens)
if num_faces > vocab_info["face_index_size"]:
    raise ValueError(
        f"Sample has {num_faces} faces, but config supports only {vocab_info['face_index_size']} face-index tokens"
    )
```

含义：

- `face_index_size=50` 表示 face index token 只能表达 `0-49` 这 50 个 face 编号。
- 如果样本超过 50 个 face，当前 holistic sequence 和重建脚本都无法直接表达它。
- 这是 token 词表设计限制，比 `max_face` 参数更根本。

## 5. Edge 数量为什么没有同样的词表容量

当前 token 序列里 edge 本身没有单独的 edge index token。每条 edge block 主要包含：

```text
src_face_index, dst_face_index, edge_bbox_tokens, edge_geometry_tokens
```

因此：

- face 数量受 `face_index_size=50` 直接限制。
- edge 数量没有一个类似 `edge_index_size=150` 的词表限制。
- 但 edge 数量会影响 sequence 长度、训练难度和后处理复杂度。
- 所以 `max_edge=150` 更像 AR 数据预处理的人为过滤上限，不是 token 词表硬上限。

## 6. Face Cyclic Re-index 的限制来源

`2sequence.py` 会对 face index 做 cyclic offset，目的是让 AR 训练时不要过度依赖固定 face 编号。

代码位置：

```text
2sequence.py:493-499
reconstruct_vqvae_sample.py:224-236
```

`2sequence.py` 中：

```python
max_faces = self.args.max_face
num_faces = num_face
N = min(self.face_index_size, max_faces)  # This is 50
r = random.randint(0, N - 1) if N > 0 else 0
face_index_map = {i: (i + r) % N for i in range(num_faces)} if N > 0 else {i: i for i in range(num_faces)}
```

当前重建实验中，为了复现实验和方便对比，常使用：

```bash
--reindex_offset 0
```

这表示不随机偏移 face index。它不会改变真实拓扑，只是让 token 编号保持直观。

## 7. 当前实验选样建议

当前 VQVAE 单样本重建实验虽然不是训练 AR，但为了让流程尽量贴近原论文的 AR+VQVAE 后半段生成逻辑，建议选样时仍然参考 AR 预处理限制：

```text
face <= 50
edge <= 150
```

原因：

- `face <= 50` 是硬约束，因为 `face_index_size=50`。
- `edge <= 150` 不是硬词表约束，但和原 AR 训练数据分布一致，后处理也更稳定。
- 超过这些范围的样本即使 VQVAE 本身能 encode/decode 单个 face/edge，也不一定能被当前 holistic sequence 和 `reconstruct_cad_from_sequence()` 顺利表达和重建。

因此，我们之前挑复杂样本时说“接近 `50/150` 上限”，更准确的含义是：

```text
接近原 AR sequence 预处理默认过滤上限，并且不超过当前 face index token 容量。
```

## 8. 小结

不同阶段的限制如下：

```text
STEP 解析阶段:
  face > 200 会被跳过。
  未看到 edge > 150 的过滤。

VQVAE 训练阶段:
  主要训练单个 face / edge 的 NCS 几何。
  不按完整 CAD 的 max_face=50/max_edge=150 过滤。

AR 序列预处理阶段:
  max_face 默认 50。
  max_edge 默认 150。
  超过会 return None，不进入 AR sequence 数据。

Token 词表阶段:
  face_index_size = 50。
  当前 sequence 最多表达 50 个 face index。
  没有独立的 edge_index_size = 150。

当前 VQVAE 重建实验:
  建议选 face <= 50、edge <= 150 的样本。
  这是为了和 AR 训练/生成逻辑对齐，并避免超过 face index token 容量。
```
